第2章    概述

2.1   背景介绍

根据中国移动集团湖北公司市场部的用户运营规划，为充分降低用户流失风险，提高运营保有成效，希望在利用有限的营销资源和机会的情况下，加强湖北移动存量用户保有运营工作的稳步开展，湖北移动和亚信联创咨询部共同携手，建设存量用户保有运营项目。

2.2   功能概述

存量用户保有运营项目以存量用户为运营目标，基于研究用户通信使用行为、粘性业务使用情况、营销合约捆绑情况等因素，分析用户通信下降和离网流失风险，以数据挖掘算法为依托，得到流失预警用户及其与之相匹配的营销策略，旨在统一的保有营销工作管理平台中，为存量用户保有运营工作提供完善的目标用户预测与营销策略引导。

 

第3章    模型设计与固化

3.1   模型逻辑结构

存量用户流失预警模型所涉及的指标维度包括两个方面：用户粘性与通信行为。

在用户粘性方面，模型使用用户资费、用户属性、交往圈、营销捆绑等字段，对用户在移动网络的稳定使用程度进行考察；

通信行为方面，流失预警模型分别从日、周、月三个时间周期上，考虑用户在不同时间周期上的通信行为性特征，对产生通信下降进而可能出现流失的用户进行预测和预警。

流失预警模型利用逻辑回归模型计算得到用户的流失概率，并对具有不同流失风险的用户进行分级预警，从而为存量用户保有工作提供支撑服务。

3.2   目标客户筛选

提取进入流失预警模型的基础用户，基础用户需满足如下几个筛选条件：

1)  入网时间<=2011年12月31日；

2)  当前用户状态为“正常使用”，或用户状态为“停机”且停机天数<=30天的用户；

3)  非随e行及行业应用用户；

同时在基础用户中剔除如下用户：

4)  合约捆绑用户：上月底已经有合约捆绑，且合约在上月底没有到期的用户；

5)  垃圾短信用户：前3个月累计有垃圾短信标签的用户；

6)  移动内部员工、数据卡用户、公免卡用户、测试卡用户、公话卡用户、机器卡用户、无线座机用户、VIP用户；

7)  SP套利用户：前一月有SP自消费标签的用户；

8)  渠道养卡用户：前一月有疑似渠道养卡标签的用户；

9)  跳蚤用户：前一月有重入网标签，且最近6个月内重入网次数>=2次的用户；

对筛选出的基础用户提取与模型计算有关的字段信息，包含每月移动用户基本资料、业务订购、消费行为及终端信息等，作为《用户基础信息表》，并由此表生成《流失预警模型数据输入表》进入流失预警模型，最终通过模型计算得到《流失预警模型数据输出表》。

   (-0.4258) * TD_FLAG +

   (-0.368) * ln_on_line +

   (-0.177) * brand_1 +

   0.3239 * brand_2 +

   (-2.592) * Post_paid_flag +

   (-0.9977) * ln_last_balance +

   (-0.05613) * ln_current_points +

   (-1.119) * ln_D1_TOTAL_CNT +

   (-0.1378) * ln_D1_TOTAL_DUR +

   0.427 * ln_D1_RM_OUT_CNT +

   + 2.61)

3.4.2     流失风险分级预警

将P值转化为用户的稳定度得分，流失风险分级预警需要根据这一得分大小进行分段，从而为用户保有工作提供帮助。

说明：对稳定度得分四舍五入，保留一位小数。

根据用户的稳定度得分，将用户划分为稳定用户和非稳定用户，并确定非稳定用户的流失预警等级：

稳定用户：稳定度得分为(6.0,10]，此部分用户的流失风险较低，稳定性较好；

非稳定用户：稳定度得分为[0,6]，此部分用户均具有一定的流失风险，根据流失风险的高低可以继续对用户划分不同的预警等级如下：

1）红色预警：稳定度得分为[0,1.0]，此部分用户的流失风险最高，是营销的重点目标用户；

2）橙色预警：稳定度得分为(1.0,3.0]，此部分用户的流失风险较高，也是营销的主要目标用户；

3）黄色预警：稳定度得分为(3.0,6.0]，此部分用户有一定流失风险，但相对前两类用户而言其风险较低，需要通过营销提高其稳定性。
