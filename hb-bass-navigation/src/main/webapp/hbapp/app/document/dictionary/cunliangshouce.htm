湖北移动存量运营工作台操作手册

1.1      手册分册

本手册为存量运营工作台系统帮助文档。

1.2      使用建议

本手册编写目的是为了引导用户使用存量保有工作台，通过本手册了解该系统涉及存量保有工作中的哪些环节。

营销首页―>客户群管理―>营销策划―>营销审批―>营销执行

使用本手册前一定保证系统已经完成初始化等配置工作，正式开始运营使用。

1.3      名词解释

l  营销活动 ―营销的一次执行称为一个活动；

l  客户群 ― 客户的集合，作为营销活动的目标客户或对比客户

l  客户群模板 － 客户群体的定义规则

l  客户群清单 － 具体的客户集合，包括两种创建方式：从通过导入文件创建、通过存量保有客户画像创建

l  客户属性 ― 客户的属性标识及相关业务信息等，如年龄、消费情况等一切和客户有关的信息，用于筛选客户和给客户分组；

l  目标客户 ― 最终选定参与活动的目标人群

l  审批流程 － 设定审批规则，如单级或多级审批，每级审批达到什么触发条件时，由哪个部门或哪个用户审批

l  审批关系 － 设定部门的审批人

第2章                    系统简介

2.1      系统功能和用途

“存量运营工作台”是由“亚信科技（中国）有限公司”开发的经营分析系统的一个子系统，主要完成存量客户运营中除营销活动执行环节之外的所有支撑工作。

存量运营工作台反映业务部门的存量保有工作流程，并对保有的过程信息进行IT化管理。存量保有工作流程包括：

客户群管理-营销策划-营销审批-营销执行

l  “客户群管理”设定存量保有营销活动目标客户；

l  “营销策划”包括设定营销的目标、使用营销渠道、目标设定以及建议套餐等信息；

l  “营销审批”主要是部门内的领导审批，支持多级审批，同时可设置通过短信通知审批人或策划人；

l  “营销执行”包括“活动派单”。

此外，可通过存量保有运营监控表监控存量保有工作执行效果。

“配置管理”包括“系统配置管理”，“营销配置管理”；例如：“审批关系定义”、“审批流程管理”、“营销活动指标定义”、“维度字段管理”、“模板管理”等等对营销管理系统进行支撑的基本管理功能。

另外系统还提供丰富的查询功能：如营销信息查询、营销执行查询、系统配置查询等。

第3章                    客户群管理

进入营销策划前，首先确定存量保有活动的目标客户。

为了简化策划过程、便于重复使用，系统将目标客户选取模块单独提出来，称为“客户群管理”，分为2大部分：

l  客户群创建 － 用于定位目标客户群，提供了几种不同的方式创建客户群，供营销活动使用

l  客户群管理－ 用来对已创建的客户群进行管理，包括修改、删除、发布和用户清单查询等功能

下面详细介绍这2个模块：

3.1      客户群创建

功能入口：存量运营－>客户群管理

默认页面为菜单列表页面，展现创建方式和相关说明。

主要两种创建方式：

l  从导入文件创建DD以一个号码清单文件为基础，按照用户定义的条件剔除掉部分号码后，生成满足要求的客户群清单。

l  存量保有客户画像创建DD稳定度模型输出客户稳定度级别，结合客户各项属性特征，生成满足要求的存量保有活动目标客户。

下面分别介绍：

点击“创建客户群”按钮，进入如下保存用户群界面。

3.1.1      从导入文件创建

选择“从导入文件创建”选项，进入如下页面：

页面信息说明：导入文件中文名称――上传文件的名称（不允许重名）。选择导入文件――选择带有客户信息的文件。文件描述――描述上传文件。

输入完信息后，点击上传按钮。

上传文件的时间和文件大小有关。

     成功导入后的目标客户群可在客户群管理中进行查询。

3.1.2      存量保有客户画像创建

点击存量保有客户画像创建。

其中，基本查询条件区域中客户稳定度级别为存量客户稳定度模型生成。

根据保有活动开展要求，对用户基本属性、用户扩展属性、用户通信行为、用户收入属性及用户捆绑信息进行设置，确定保有活动目标客户。

客户属性可通过下拉框或填写方式进行选择。

选择各项属性后点击查询。

点击生成目标客户群

填写客户群名称，选择营销活动类型，并完成客户群描述，点击确定。

生成后的目标客户群可在客户群管理中进行查询。

3.2      客户群管理

点击客户群管理

可通过选择填写客户群名称、所属地市、创建人三项信息，查询已创建的客户群。

点击客户群名称前方框选择客户群，点击删除即可删除当前用户创建的目标客户群。

 

第4章                    营销策划

功能入口：营销策划，进入后，主页面显示的是当前用户有权限进行活动策划的营销案列表，如下：

 

4.1.1      营销活动策划

选择营销活动策划（存量）进入策划页面

填写存量保有活动信息点击确定，系统提示营销活动创建成功

在营销活动管理中，可查询到创建的存量保有营销活动。

4.1.1.1     活动信息定义

l  营销活动名称：活动名称；

l  活动启动日期：该活动计划开始执行的日期，不能早于营销案启动日期；

l  活动结束日期：该活动计划结束的日期，不能晚于营销案结束日期；

l  目标客户群：可通过下拉框选择当前用户创建的目标客户群。

l  营销成本：该活动营销活动预计使用营销成本金额。

l  目标客户筛选：该活动目标客户筛选的描述。

l  是否PUSH推荐：如选择PUSH推荐后，需填写营销用户。如选择否，则需勾选其它营销渠道，并填写对应渠道的营销用语。

以上为必填项目。

4.1.1.2     提交审批

点击营销活动管理，

选择填写营销活动信息后，点击查询

可点击营销活动名称选择营销活动，点击活动详情可查看活动信息。点击提交审批，活动进入审批流程，审批完成后自动提交确认。

4.1.1.3     删除活动

策划状态的活动，和没有审批记录的终止状态可以删除，选中要删除的活动，点击右上角的“删除”按钮。

注意：一旦活动进入了“审批”、或者“执行”状态，该营销活动将不允许删除。

第5章                    营销审批

本章将主要介绍审批相关的各种操作。

选择营销活动审批

设置查询条件后，选择查询。勾选单个营销活动，点击审批

5.1      审批结果说明

系统提供三种审批结果：

l  同意DD代表您同意活动的策划内容，审批通过；

l  需调整DD代表您认为有些地方需要策划人再调整一下，再提交审批；

l  策划终止 DD代表您认为此活动没有营销价值，终止活动；

如果点击这些按钮时，提示“此活动已审批过”，说明当前审批人已通过其他方式审批过，无需再次审批。

注：如需多级审批，只有下级审批人员审批通过的情况下，上级审批人员才会看到审批的活动, 不能进行越级审批。选择了需调整和审批终止时，会直接发给策划人消息进行调整或终止活动。调整后报批的营销案或营销活动会根据审批流程的设定，重新进行逐级审批。

5.2      营销活动审批

审批人有以下方式获得是否有待审批的营销活动。

1、登陆系统查看是否有待处理的营销案。

操作步骤是登陆系统－>营销审批－>营销活动审批。可看到待审批和已审批的营销活动列表：

系统提供批量审批和逐个审批两种方式。

l  批量审批：如果同时审批多个营销活动，可直接选中要审批营销活动左边的复选框，点击审批意见按钮，进行批量审批。批量审批不能填写额外的审批意见。

l  逐个审批。

2、除登陆系统查看外，当有需要审批的营销案时，审批人会收到短信通知，这时就可以登陆营销管理系统的审批模块进行活动审批。

5.3      审批委派

系统针对由于出差或其他原因无法正常进行活动审批的审批人，提供了审批委派功能。

功能入口：营销审批－>审批委派

界面如下：

只需点击“选择”按钮，选择被委派人，然后点击“委派”按钮即可委派，在取消委派之前，所有提交由委派人审批的活动都会自动转到被委派人处审批。

如果委派结束，只需进到这个界面，点击“取消委派”按钮就可以收回审批权。

5.4      审批状态查询

策划人提交审批后，如需了解审批阶段，系统可提供多种方式通知审批人。

5.4.1      营销活动的审批进度查询

进入营销活动策划―>营销活动管理

可选择营销活动状态等选项，查询营销活动

勾选单个营销活动，点击审批日志，可查询活动审批进度。

如果开通短信功能，策划人将收到审批结果的短信。

 

 

第6章                    营销执行

执行菜单的功能入口：营销执行。

 

6.1      营销活动派单

选择营销派单，如下图，选择需要派单的营销活动，点击“派单”按钮，完成活动的派单。

选中营销活动，点击派单

由于一个营销活动的策划、审批在经分侧，而真正进行用户推荐的环节在boss或客户侧，当派单后，系统会生成接口文件传送到boss或客户系统。

6.2      营销活动传送BOSS监控

点击营销活动传送BOSS监控，可对营销案在系统间传送结果进行监控。

点击图中查看详细按钮，进入营销任务统计查询页面，如下图所示，选择统计查询条件，点击“任务统计”按钮，页面下半展示栏展示统计数据。

点击存量运营监控报表，可查看存量运营相关指标发展情况。
