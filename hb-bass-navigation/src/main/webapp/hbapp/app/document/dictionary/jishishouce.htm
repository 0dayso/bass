1.   建设的目标

²  提升地市数据集市对一线生产运营支撑的能力。

²  深化数据及信息业务数据集市的分析支撑能力。

²  提升集团客户数据集市对集团客户业务运营管理的分析支撑能力。

2.   建设原则

A.   唯一性原则：省级经营分析系统数据仓库是数据集市的唯一数据来源，数据集市的源数据皆来自省级经营分析系统数据仓库。

B.   标准化原则：数据集市的逻辑模型要求遵照经营分析系统数据仓库逻辑模型规范中有关数据集市的相关要求进行构建。

C.   统一访问原则：数据集市各项应用展现功能必须集成在经营分析门户中，并通过该门户进行访问。

D.   功能复用原则：数据集市各项应用展现功能必须集成在经营分析门户中，并通过该门户进行访问。

E.   整合集中原则：数据集市为业务部门及地市分公司提供统一和完善的分析管理支撑,各业务部门及地市分公司的外围分析系统应逐步整合到数据集市中。

F.   专业化原则：按照分析专业化的原则，将各业务部门及地市分公司需求和应用转移到地市数据集市、数据及信息业务数据集市和集团客户数据集市上。

3.1.3.2.3       参数和数据源的混合设置

参数和数据源的混合设置，分两种情况；

²  在参数的“条件脚本”中设置；

     在参数的“条件脚本”中，可以加入数据源中的条件语句。如：在参数para1中的条件脚本填写---USERID={para1}，这里的USERID为数据源中的字段。

   在报表配置完成后，当该参数有值时，点击查询按钮，此条件脚本语句将添加到数据源的SQL语句中。

²  在数据源定义时直接引用参数；

     可以在数据源定义中直接引用已经设置的参数，所引用的参数应设置好默认值。

   例如：

    select REGION,AREA_ID,AREA_NAME,ENABLE,DBNAME,TBSPACE from md.dim_area where area_id='{para1}‘

  如果参数所关联的字段为字符型字段，需要加入单引号。

备注：

    情况一，适用于报表开始时显示全部记录，使用者在选择条件后进行重新筛选等情况。

    情况二，适用于报表开始时显示默认情况的记录，适用者在选择条件后重新筛选等情况。

3.3.4.4         常见逻辑实现

A．二元逻辑：IF…ELSE…

B． 条件逻辑：if(条件) then .. else if（条件） then .. else …

目前不支持条件逻辑表达，但条件逻辑表达可以转成二元逻辑表达来实现。

 

C． 循环。While (条件) do…

定义数组格式：变量名;select语句

例如

 &array_ctrl;select distinct DB_NAME,

                OBJ_TYPE,

                OBJ_NAME,

                ltrim(rtrim(CYCLE_ID)) as CYCLE_ID,

                case

                  when length(rtrim(cycle_id)) = 6 and obj_type = 'INTER' then

                   'M'

                  else

                   'D'

                end as cycle_type

  from DQ.CFG_CHECK_CTL

 where check_flag = 1 with ur;

l  取数组当前行的第n列的值格式：数组名.get(n)

例如 取&array_ctrl数组的第一列的值 &array_ctrl.get(1)

l  可同时在一个节点内进行多个操作

例如 变量定义：

&OBJ_SOURCE;&array_ctrl.get(1);0;

&OBJ_TYPE;&array_ctrl.get(2);0;

&OBJ_NAME;&array_ctrl.get(3);0;

&CYCLE_ID;&array_ctrl.get(4);0;

&CYCLE_TYPE;&array_ctrl.get(5);0;

类似执行多条sql语句，可同时在一个节点内完成多个变量的定义。

l   数组步长计算格式：+;1;数组名.next();

例如&array_ctrl数组步长加1操作：+;1;&array_rule.next();

判断数组是否访问完毕格式：数组名.next();数组名.eof();<;

例如：&array_rule.next();&array_rule.eof();<;

通过数组动态定义，实现了嵌套循环

 

3.3.5   调度任务

可以将ETL中设置的程序，加入到本调度程序中。

 

介绍如下；

编号：为此调度程序的唯一标识；

名称：为此调度程序的名称；

所属地区：系统自动填写，为用户所属地市的编码；

组：保留字段；

对应程序名：填写ETL程序的编码名称；

周期类型：为调度的周期种类；

月第几天：当周期类型为“月”时，需要填写；

下次执行时间：可以显示已经加入调度程序的下次执行时间，但当设置时，需要设置为小于当前时间的任何时间；

手工调度批次：当需要手工调度程序时，指定运行的批次号；

命令参数：不可编辑，在保持该配置时，系统会自动填写相关命令参数；

状态：是否开启调度程序；

此页面包含两种逻辑方式；

ü  自动调度

当调度任务中设置的ETL程序后，ETL程序所填的当前批次号设置为所需的批次号，并将当前调度任务设置为“纳入调度”，点击“保存”按钮，即可将该ETL程序纳入后台自动调度任务。

ü  手工调度

当调度任务中设置的ETL程序，并在手工调度批次中填写了所需的批次号，周期类型中设置为“手动”，并将当前调度任务设置为“纳入调度”，点击“保存”按钮，即可将该ETL程序按照设置的批次号立即执行。
